# トラブルシュート

## エンジンが選べない / 使えない

画面下部の **Backend status** に各バックエンドの状態と理由が表示されます。「not available」の場合は、重みや実行ファイルが未配置か、依存パッケージが未インストールです。[モデルのセットアップ](setup/models.md) を確認してください。

## メモリ不足（OOM）で落ちる / 失敗する

- `Tile` に 512 や 256 などを設定して、タイル分割で処理してください（既定の 0 は一括処理）。
- `Half precision` を ON にすると省メモリになります。
- `Scale` を下げると出力解像度が小さくなり、必要メモリも減ります。

## 色がおかしい / NaN が出る（Spandrel）

`Half precision` を OFF にして再試行してください。MPS で半精度が不安定な場合があります。

## 出力が表示されない / ファイルを開けない

- デスクトップ版では任意のパスの画像を扱えるよう設定していますが、極端に大きい画像は表示生成に時間がかかることがあります。
- 入力パスに日本語が含まれる場合、選択は NFC へ正規化して扱います。それでも開けない場合は、英数字のみのパスで再試行してください。

## デスクトップ版（Tauri）が起動しない / 画面が出ない

- 初回起動時は Python サーバの立ち上げに数秒かかります。
- `.venv` が見つからない場合は、`UPSCALAR_PYTHON` 環境変数で Python 実行ファイルを明示できます。
- ウィンドウを閉じると Gradio サーバも停止します。アプリがクラッシュ／強制終了した場合も、Python 側が親プロセスの消滅を検知して自発終了します。

## ファイル/フォルダ選択ダイアログが開かない

- ブラウザ版（`python app.py` を直接起動）では、ネイティブダイアログは使えません（テキスト手入力にフォールバックします）。
- デスクトップ版（Tauri）で開かない場合は、`src-tauri` を再ビルドして最新の権限設定（capabilities）が反映されているか確認してください。

## ポートが使われている

`7860` が使用中の場合は、近い空きポートで自動起動します。固定したい場合は `UPSCALAR_PORT` を指定してください。

```bash
UPSCALAR_PORT=7870 python app.py
```
