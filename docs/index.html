Upscaler ユーザーガイド¶

Upscaler は、ローカル環境で動作する画像アップスケーラーの実験用 GUI です。非商用利用を前提に、軽量な確認用バックエンドから Apple Silicon の Metal (MPS) を活かす高品質バックエンドまでを、同じ画面から切り替えて試せます。

  • ソースコード / 配布: GitHub リポジトリを参照してください。
  • 動作形態: ブラウザ版（Gradio）と、ネイティブウィンドウのデスクトップ版（Tauri）の両方で同じ UI が使えます。

このガイドの使い方¶

  • まず動かしたい場合は はじめに を読んでください（インストール・起動・デスクトップ版アプリ）。
  • どの設定が良いか迷ったら、画面上部の「プリセット」から選んで試すのが近道です。 詳しくは パラメータ設定 › まずはプリセットから を参照してください。
  • 各パラメータの意味を知りたい場合は パラメータ設定 を参照してください（このガイドの中心です）。
  • どのバックエンドを選ぶべきか迷ったら バックエンド一覧 を参照してください。
  • モデルや実行ファイルの入手は モデルのセットアップ にまとめています。
  • うまく動かない時は トラブルシュート を確認してください。

主な機能¶

  • 単体画像 / 複数画像（バッチ） / ディレクトリ入力のアップスケール
  • 入力・出力の横並び比較画像の生成（*.compare.png）
  • ジョブ ID 付き出力ファイル名による上書き防止
  • 出力ごとの設定マニフェスト（*.png.json）とバッチ実行レポート（batch_report_*.md）
  • 実行履歴（outputs/history.jsonl）と履歴ギャラリー
  • バックエンド検出・診断表示
  • 実行中の Stop による途中キャンセル
  • 設定の永続化（user_prefs.json）— エンジンやスケール等を次回起動時に復元
  • ネイティブのファイル / フォルダ選択ダイアログ（デスクトップ版）
  • CLI 実行（python -m upscaler.cli）

用語¶

用語 意味
バックエンド / エンジン アップスケールを実行する実装。Pillow / Real-ESRGAN / Spandrel / A-ESRGAN / External Command の 5 種類。
スケール (Scale) 出力の拡大倍率。
タイル (Tile) 大きな画像をタイル分割して処理する単位（メモリ対策）。
ジョブ ID 出力ファイル名に付く一意な識別子。上書きを防ぐ。
